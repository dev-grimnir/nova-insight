nova-insight

An ISP support dashboard that lives on top of your RADIUS server.

Live demo. Production code, synthetic data. Source: github.com/dev-grimnir/nova-insight

What this is. nova-insight is a userscript that overlays a customer-health dashboard on top of a RADIUS server admin interface. It pulls connection event history, classifies links by stability, surfaces problems before customers call, and produces reports suitable for handoff to field techs or attachment to a ticket.

Why it exists. The native interface presents connection history as a flat spreadsheet of timestamps. Reading a customer's stability means eyeballing session lengths and counting records inside a time window — doable for one customer, impractical for many.

This project started as a single-purpose userscript that did one thing: fill in how long a modem had been offline after it reconnected. That script is still in scripts/. A second script came next, generating reports that looked a lot like what the Report view produces today — but it wasn't headless, and you had to babysit it.

The dashboard is what happens when those scripts grow up. There's no real cap on how many modems it can watch concurrently — once a customer is located, each poll only fetches events since that customer's last known status change, so the per-poll cost stays flat as the watchlist grows. Real-time monitoring, snapshot history, threshold-based alerting, and report generation as a side effect rather than the main event.

What you're looking at. Everything in the panel is production code. One file swap routes data calls to a synthetic event-stream generator instead of a real RADIUS server, so the 17 customers in the demo are scripted scenarios: stable residential links, flaky ones, a few business-grade nodes, a centerpiece outage in progress. The encrypted localStorage is real — set a passphrase from the admin menu and your session resumes when you come back. Generate reports, adjust thresholds, dig into snapshot histories. Nothing you do here touches a real network.

Under the hood. Vanilla JavaScript, MVC, no framework. Tailwind v3.4 (precompiled), Chart.js for snapshot graphs, AES-GCM via WebCrypto for local persistence.
