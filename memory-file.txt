## Recent main-branch work (this conversation)

### Account Not Found bug fix
**File:** `src/controllers/neonova-dashboard-controller.js`, `updateCustomerStatus` method.

**Bug:** Long-idle customers (no Start/Stop events in any lookback window up to 335 days) were being stamped 'Account Not Found' on every poll, overwriting their real Connected status. Took a status flip OR remove-and-readd to fix. The buggy block was:

    if (!latest) {
        customer.update('Account Not Found', 0);
        return;
    }

**Fix:** Guard on `customer.lastEventTime` first. Known existing customers preserve status and bump duration; only never-seen customers get stamped 'Account Not Found':

    if (!latest) {
        if (customer.lastEventTime !== null) {
            const eventDate = new Date(customer.lastEventTime);
            if (!isNaN(eventDate.getTime())) {
                const durationSeconds = Math.floor((Date.now() - eventDate.getTime()) / 1000);
                if (durationSeconds >= 0) customer.update(customer.status, durationSeconds);
            }
        } else {
            customer.update('Account Not Found', 0);
        }
        return;
    }

**Architectural note (deferred):** The buffer-update block now ingests events into `customer.eventHistory` BEFORE the status-pill logic runs. The status-pill is doing redundant HTTP via `getLatestEntry` that overlaps the buffer's data. Future cleanup: derive status purely from `customer.eventHistory` instead of refetching. Bigger reshape, separate cycle.

### Tab dead-zone bug fix
**File:** `src/views/neonova-dashboard-view.js`, `renderTabBar` method.

**Bug:** Clicking the lower half (or any padding/non-text area) of an inactive tab did nothing. Tab-switch handler was attached to `.tab-label` span only. Padding, counts span, and any non-text pixels of the button were dead.

**Fix:** Move the click handler from `.tab-label` to the button itself. Inner glyphs (mode, bell, close) already call `stopPropagation()`, so they're safe. Counts span (no handler) now bubbles to button → switches tab.

    btn.addEventListener('click', () => {
        this.controller.getTabController().switchTab(tab.label);
        this.renderTabBar();
    });

The label dblclick (rename) stays where it was.

### Column sort glyph layout
**File:** `src/views/neonova-dashboard-view.js`, `buildTheadHTML` method + style block.

**Bug:** Sort glyphs (⇅/↑/↓) sat above narrow column labels instead of inline-left. Friendly Name, Radius UN, Status wrapped to two lines with glyph alone above; Duration was short enough to stay inline.

**Fix:** Wrap glyph + label in `.th-content` flex container. Plus thead vertical-align fix.

Markup change in sortable branch:

    return `<th class="px-2 py-2 text-${c.align}">
        <span class="th-content">
            <span class="${glyphClasses}" data-column="${c.key}" title="Sort by ${c.label}">${glyph}</span>
            <span class="column-label">${c.label}</span>
        </span>
    </th>`;

CSS added to style block:

    .th-content {
        display: inline-flex;
        align-items: baseline;
        gap: 6px;
    }
    .neonova-dashboard thead th {
        vertical-align: bottom;
    }

`.sort-glyph` rule was duplicated in the style block (margin-right: 6px in old, flex-shrink: 0 in new). Duplicate cleaned up — kept the new version, removed margin-right since `.th-content` owns spacing via gap.

### Header button uniformity
**File:** `src/views/neonova-dashboard-view.js`, `getHeaderHTML` + `updatePrivacyButton`.

All 5 header buttons (Privacy, Admins, Pause Polling, Refresh, Add Customer) unified to single shared class string:

    whitespace-nowrap px-4 py-2.5 bg-emerald-700 hover:bg-emerald-600 text-white font-medium rounded-2xl flex items-center justify-center gap-2 transition-all border border-white/40 shadow-sm

- Emerald-700 fill matches active tab color (visual family)
- White text, white/40 thin border
- `whitespace-nowrap` prevents two-word labels from wrapping
- `px-4` (was `px-6`) to save horizontal real estate

Privacy button text changed to verb form: "Enable Privacy" / "Disable Privacy" (was "Privacy On" / "Privacy Off"). Action-on-click convention, matches imperative tone of other buttons. Title attributes carry current state info.

Pause Polling icon and interval suffix recolored from emerald to white/white-70 to match new fill.

**Pause Polling vertical-fatness fix:** The `text-sm` class on the interval suffix span was forcing a font-size mismatch with the main label, producing extra line-height. Removing `text-sm` made all five buttons match in height. The `text-white/70` opacity still makes the suffix read as secondary.

### Privacy button gotcha
`updatePrivacyButton` overwrites className on every call (including initial load). Its className string is a SECOND copy of the button class — must be kept in sync with `getHeaderHTML`. Easy to miss when bulk-editing. This is a strong argument for the upcoming shared button class refactor.

## Debugging episodes worth remembering (this conversation)
- web_fetch returned stale main TWICE during the Account Not Found diagnosis. The user pasted current source; my patch was based on stale code that referenced a removed `showToast` method. Always ask the user to paste current source before writing edits against a class with non-trivial existing logic.
- Silent CSS failures: a CSS edit was added to the source file but didn't appear in the deployed style block. Diagnosis required dumping `document.querySelectorAll('style')` from the console to inspect what was actually present. The `.th-content` rule was missing entirely; `.sort-glyph` was present but duplicated. Don't trust source-matches-deployed assumption — verify with a console dump.
- The `if (!document.getElementById('neonova-scroll-style'))` guard in `createElements` is a latent risk: if a userscript reload encounters an existing style block, new CSS rules don't get re-injected until a hard reload clears the old block. Worth fixing in a future cycle (replace contents instead of skipping).
